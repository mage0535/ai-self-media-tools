# 内容文案风格指南

## 默认生成工作流

所有内容生成任务默认遵循这条链路：

1. 先检索同赛道热门内容、潜在热门内容、近期爆款内容
2. 分析参考样本的：
   - 标题结构
   - 开头钩子
   - 排版节奏
   - 情绪密度
   - CTA 方式
   - 图文风格
   - 视频节奏
3. 结合趋势阶段给出本次内容角度：
   - `hot`：热点深度分析
   - `emerging`：潜力趋势追踪
   - `viral_candidate`：爆款信号拆解
4. 再生成：
   - 主标题
   - 主文案
   - 开头钩子
   - 行动号召
   - 图片提示词
   - 视频提示词
   - 推荐 hashtags

默认要求：

- 文案必须吸收同赛道爆款的节奏感，但不能直接洗稿
- 图文要有可视化切入点，不能只有纯文字
- 视频脚本必须先给钩子，再给 3 个关键点，最后给收束动作
- 有来源时必须优先用来源支撑事实，不得凭空编数字

## 趋势驱动规则

- 热点内容：优先用“热点-解读-预判”结构
- 潜力趋势：优先用“问题-方案-行动”结构
- 爆款信号：优先用“清单-深度-总结”结构
- 同一赛道连续 7 天内避免重复标题结构和重复结论句

## 图文与视频联动规则

- 图文封面提示词要提炼：
  - 主题对象
  - 视觉冲突
  - 对比关系
  - 信息密度
- 视频提示词要提炼：
  - 首句钩子
  - 3 个展开点
  - 收尾 CTA
  - 目标平台语气

## 爆款参考规则

- 参考样本优先取同赛道、同语种、同平台或相近平台
- 至少抽取 2 条参考内容后再定调
- 若无参考样本，则退化为赛道默认模板，不允许空白生成

## 核心原则

### 标题法则
- **字数**：15-28 字最佳，超过 30 字在手机端被截断
- **关键词前置**：核心词在前 15 字内
- **情绪触发**：每标题至少包含一个情绪词（警惕、曝光、实测、揭秘、重磅）
- **受众匹配**：标题让目标读者一眼认出"这是写给我的"
- **搜索适配**：包含搜一搜高频关键词
- **至少准备 3 个备选标题**，用标题评分工具初筛，最终人工选定

### 排版「三三原则」
1. **每段 ≤3 行**（手机端显示）：密密麻麻的长段像一堵墙，读者直接划走
2. **每 3 段插一个视觉元素**：图片、引用块、表格、分割线均可
3. **关键信息用 3 种方式强调**：加粗、emoji 标记、分段突出（不要滥用）

### 内容结构模板

#### 模版一：问题-方案-行动（适合教程/方法论）
```
开头：抛出痛点/问题
中间：系统解决方案（分步骤/分层次）
结尾：总结要点 + 行动号召
```

#### 模版二：故事-观点-延伸（适合案例分析）
```
开头：真实故事/案例（降低防御心）
中间：自然引出观点
结尾：延伸到读者自身场景
```

#### 模版三：热点-解读-预判（适合追热点）
```
事实层：还原事件核心事实（表格对比）
分析层：独特视角解读
洞察层：提炼规律 + 趋势预判
时效性要求：24-48 小时内
```

#### 模版四：清单-深度-总结（适合干货合集）
```
全貌表格：让读者快速扫描全局
逐项展开：深度解读每个要点
行动清单：收尾引导收藏/分享
```

### 视觉编排规范

#### 表格（高频使用）
- 用于对比数据、特性列表、时间线、优劣势
- 表头行带浅灰背景 (`#f5f7fa`)
- 保持 2-3 列，避免过宽
- 表格前后用空行+分割线与其他内容区隔

#### 引用（`>` 语法）
- 关键引文使用 `>` 引用
- 每篇文章 1-3 处引文，分布在不同位置
- 引文内容必须是高信息密度的金句或核心结论

#### 分割线 `---`
- 用于段落间的视觉区隔
- 每篇文章 2-4 处分隔线
- 分割线后紧跟新一节的小标题

#### 列表
- **无序列表**：用 `-`，3-5 项为佳
- **有序列表**：用 `1.`，适用于步骤说明
- 列表项首字母大写（中文正常标点）
- 列表项长度控制在 20 字以内最佳

#### 加粗 `**bold**`
- 全文加粗比例控制在 2-5%
- 哪些内容必须加粗：
  - 核心数字/数据（如"估值达到 **5 亿美元**"）
  - 关键结论句
  - 对比中的胜出一方
  - 技术术语首次出现

### 语言风格

#### 科技号标准语态
- **专业但不晦涩**：用行业术语，但首次出现时附带简要解释
- **观点鲜明**：每个小节有明确立场，避免泛泛而谈
- **数据支撑**：重要判断附带数据/案例
- **简洁有力**：删掉"笔者认为""值得注意的是"等废话填充词

#### 每篇文章必备元素
1. 开头 **3 秒钩子**：第一个段落必须抓住注意力
2. 每 **300 字设一个钩子**：新数据、反直觉观点、关键引文
3. 结尾 **行动号召**：总结升华 + 开放性问题/引导互动
4. 至少 **1 张数据/对比表格**
5. 至少 **1 处引用块**
6. 至少 **2 处分隔线**

### 平台差异

#### 微信公众号（wechat）
- 标题：≤64 字
- 摘要：≤120 字（纯文本，取自正文前 120 字）
- 正文：完整 HTML，支持标题/列表/表格/图片/引用/代码块
- 封面：需要单独上传图片
- 支持评论区互动

#### 小红书（xiaohongshu）
- 标题：≤20 字
- 正文：≤950 字（纯文本，去掉所有格式）
- 话题标签：6 个以内，从 topic 中提取关键词
- 图片优先：1 图 + 短文案模式
- 语气更偏向"朋友推荐"，使用"种草""安利"等平台用语

### 技术实现

内容正文使用 Markdown 语法编写，`format_for_platform()` 会自动将 Markdown 转换为对应平台的格式（HTML/纯文本等）。

支持的 Markdown 语法：
- `#/##/###` 标题 → `<h2>/<h3>` 标签
- `**bold**` → `<strong>` 加粗
- `*italic*` → `<em>` 斜体
- `` `code` `` → `<code>` 行内代码
- `` ``` `` 代码块 → `<pre><code>` 代码块
- `- item` → `<ul><li>` 无序列表
- `1. item` → `<ol><li>` 有序列表
- `> text` → `<blockquote>` 引用
- `| col1 | col2 |` → `<table>` 表格（带样式）
- `---` → `<hr>` 分割线
- `![alt](url)` → `<img>` 图片
- 普通段落自动包裹 `<p>` 标签
- 所有文本自动 HTML 转义

### 实用提示

- **AI 受限**：平台对机器生成内容有检测机制，核心观点/案例/情感必须人工注入
- **质量优先**：推荐周期可达 7 天，频繁更新反而会冲掉正在推荐的内容
- **互动引导**：结尾抛出问题引导评论区互动，提升算法权重
- **社交关系**：社交分享占算法权重的约 35%，培养铁粉分享至关重要
